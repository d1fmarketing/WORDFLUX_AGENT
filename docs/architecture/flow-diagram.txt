KEY PRINCIPLES:

1. SEPARATION OF CONCERNS:
   - Layer 1 (Chat): Pure request/response, no side effects
   - Layer 2 (Queue): Durable storage, async processing
   - Layer 3 (Worker): Actual execution, side effects allowed
   - Layer 4 (Cockpit): Redis primitives, no HTTP
   - Layer 5 (Frontend): Read-only, reactive updates

2. ASYNCHRONOUS FLOW:
   - Chat returns immediately with job_id
   - Worker processes job in background
   - Frontend updates via SSE (no polling)

3. NO TIGHT COUPLING:
   - Chat doesn't know about Redis schema
   - Chat doesn't depend on board availability
   - Worker can scale independently
   - Frontend doesn't poll for changes

4. AUDIT TRAIL:
   - All jobs logged in wf:ledger:{job_id}
   - All events stored in wf:events:recent
   - Full history for debugging

5. FAILURE HANDLING:
   - Jobs can be retried from queue
   - WIP limits prevent bottlenecks
   - Circuit breakers for external APIs
   - Graceful degradation
